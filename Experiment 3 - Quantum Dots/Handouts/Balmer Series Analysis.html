Focus on one peak¶

Instead of fitting all peaks at once, I'm going to focus on one peak by creating a new dataframe that has just the data surrounding one peak.

The lines in the Balmers series of hydrogen are given Greek letter names. The bright red line at around 656 nm, corresponding to the 3-to-2 transition, is the smallest energy (longest wavelength) transition, and is called the H-alpha line. The aqua line at around 486 nm, corresponding to the 4-to-2 transition, is called the H-beta line. And so it goes, up the series to ever greater energies and smaller wavelengths.

Here I'm going to zoom-in on the H-$\beta$ line. I'm going to create a new dataframe called Beta that contains the data surrounding this peak. Eventually, I'll fit a peak function to this data.

Define a model function to fit the data¶

Now I want to define a model function to fit the data. Standard spectroscopic peak functions include gaussian, lorentzian, and voigt functions. You might be interested in researching these functions and the shape of peak functions in spectroscopy, to learn what can be said about why a peak might have one shape or another.

Here I'm going to use a gaussian function for the model. I will add to the gaussian the function for a line. the line part of the model will be used to fit the background counts, the gaussian part of the model will be used to fit the peak.

But there is something interesting about this peak. Notice that it is not symmetric. On the right hand side there is that little "kink" in the peak. You might call it a "shoulder." Such a shoulder suggests that this peak is actually a combination of at least two peaks that overlap. A taller peak at a slightly shorter wavelength (the H-$\beta$ line) and a smaller peak at a slightly larger wavelength. There are lots of reasons why there might be two peaks, and you should ask "What's the reason?" in this case.

So, instead of a single gaussian, my model function will be the sum of two gaussians (and a line). There will be eight free fit parameters: the slope and intercept of the line, and the height, width, and location of each peak. Here is my model function:

$$I(\lambda) = a_0 + a_1 \lambda + b_0 e^{-(\lambda - b_1)^2/2b_2} + c_0 e^{-(\lambda - c_1)^2/2c_2} $$

The parameters $b_0$ and $c_0$ set the heights of the peaks, the parameters $b_1$ and $c_1$ set the locations of the peaks, and the parameters $b_2$ and $c_2$ set the widths of the peaks. Great!

I define this function in code below. I'm using the generic symbol "x" for the wavelength, because it's the independent variable here.

In [50]:
def Two_Gaussians_And_Line(x, a0, a1, b0, b1, b2, c0, c1, c2):
    
    linear = a0 + a1*x
    
    exponent1 = (-0.5)*((x-b1)/b2)**2
    gauss1 = b0*np.exp(exponent1)
    
    exponent2 = (-0.5)*((x-c1)/c2)**2
    gauss2 = c0*np.exp(exponent2)
        
    return  linear + gauss1 + gauss2

Find a good initial guess¶

This is a fairly complicated model. We will have to supply the fitting algorithm a good initial guess for all of the parameters if it is to converge on the correct best fit. To find a good initial guess, I'm going to use my interactive plotting program. This is something we used a few weeks back in a Zoom class. It graphs the data and model, but has sliders for all the free parameters. By adjusting the sliders, you can shape the model so that is is a reasonably good fit, by eye, and then record all the parameter values to use as the initial guess.

The program, called "interactive_plot.py" must be copied into the same folder that has your jupyter notebook. Once copied there, you can load the program into you notebook with the following line:

In [30]:
import interactive_plot as ip

Now I am going to run the program. When you use it for the other peaks, use the code below as a template.

Basically, you have to define the seven arguments that are passed to the "slider_plot" function, which is called in the last line. They are named: 1) fitparams, 2) function, 3) plotparams, 4) xData, 5) yData, 6) xErr, 7) yErr. In turn:

1) fitparams is a list of sublists. Each sublist provides specs for a slider corresponding to one of the fit parameters. Because we have eight fit parameters, we must supply eight sublists. Each sublist contains, in order: i) the name of the parameter in quotes, ii) the slider's starting value, iii) the slider's ending value, iv) the slider's initial value, v) and the slider's step size.

2) function is the name of the defined model function. In this case, Two_Gaussians_And_Line.

3) plotparams sets the scales for the plot. In the 'x' sublist give a starting and ending value for the range of the horizontal scale in the plot. In the 'y' sublist give a starting and ending value for the range of the vertical scale in the plot. Obtain values for these ranges by inspecting the plot of the data above. The third and final sublist, just a single number, gives the number of points to generate for the plot.

4) xData is the name of the array that contains the "x" values of the data set; in this case, the wavelength column of the Beta dataframe.

5) yData is the name of the array that contains the "y" values of the data set; in this case, the intensity column of the Beta dataframe.

6) xErr is the name of the array that contains the uncertainties in the "x" values of the data set; in this case, there are none, so specify None. If an array is specified, this simply instructs the program to show the errorbars on the plot.

7) yErr is the name of the array that contains the uncertainties in the "y" values of the data set; in this case, there are none, so specify None. If an array is specified, this simply instructs the program to show the errorbars on the plot.

In the fitparams line it is important to give reasonable values. This requires some understanding on your part of how the parameters affect the shape of the model function, and the ability to define a reasonable range and initial value based on the data.

You should inspect my fitparams list carefully, and understand why my choices make sense. If you don't come to an understanding of this, you should talk with me in class about my choices. You'll need to be able to do something similar for the other peaks.

Fit the model to the data¶

Here I call the curve_fit routine to fit the model to the data. Note the four arguments that are passed: i) the name of the defined model function, ii) the array containing the wavelengths, iii) the array containing the intensities, and the key-word argument (called a "kwarg" in python) p0= that points to the list "guest" containing the initial guess for the eight free parameters.

The curve_fit function returns two arrays: i) popt contains the list of best fit values for all eight parameters, ii) pcov contains the covariance matrix (8x8 in this case). The eight values along the diagonal contain the squares of the uncertainties in each parameter, in order, from upper left to lower right.

In [53]:
popt, pcov = curve_fit(Two_Gaussians_And_Line,Beta['W'],Beta['I'],p0=guess)

Here are the best fit values, in order: a0, a1, b0, b1, b2, c0, c1, c2. Notice the locations of the two peaks, contained in b1 and c1. The main one is at 485.6 nm, and the smaller one is at 485.9 nm.

The (square of) the uncertainty in b1 is contained in the row-4 (labeled 3 due to counting from zero), column-4 (labeled 3 due to counting from 0) element of the this 2D pcov array; that is, 2.652e-6 nm$^2$. Taking the square root gives the uncertainty. In code, this can be calculated like this:

This is in nm. Since the most significant digit in the uncertainty is a 1, we'll round this to two digits. So the final uncertainty, for reporting, is 0.0016 nm. The least significant digit is in the ten-thousands place, so the wavelength of the peaks is rounded to that place as well, giving 485.6069 nm, for a final result of

$$\lambda_{\beta} = 485.6069 \pm 0.0016\,\mathrm{nm}.$$
